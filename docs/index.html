Abstract

Este projeto é uma solução, desenvolvida em C++, projetada para centralizar e simplificar a administração completa de um estúdio de yoga. O sistema oferece uma interface de console (CLI) para gerenciar de forma intuitiva todas as entidades e operações do dia a dia, desde o cadastro de alunos até o agendamento de aulas.

O pilar do desenvolvimento foi a Programação Orientada a Objetos (POO). Conceitos avançados foram aplicados para garantir um código limpo, modular e extensível:

  • Herança: Utilizada para criar hierarquias de classes lógicas, como a classe base Pessoa que define atributos comuns para Praticante e Instrutor.
  • Polimorfismo: Implementado de forma crucial no sistema de aulas, onde a classe base abstrata Aula permite ao sistema gerenciar diferentes tipos de aulas (como HotYoga e YogaPets) de maneira uniforme.

Adotando práticas de Clean Architecture, o projeto foi rigorosamente modularizado em camadas de responsabilidade distintas, refletidas na estrutura de pastas:

  • models/: Definição das entidades centrais (as classes Plano, Aula, Praticante, etc.).
  • services/: O "cérebro" da aplicação (StudioManager), controlando a lógica de negócio e o fluxo do menu.
  • data/: A camada de persistência. O DataManager é um serviço estático responsável por salvar (serializar) e carregar (desserializar) os dados dos vetores em arquivos .dat, garantindo que as informações sobrevivam entre as execuções.
  • utils/: Uma camada de utilidades de UI, como o Color.h, para o polimento da interface de console (TUI).

O resultado é um sistema de gerenciamento funcional, que não apenas implementa um CRUD (Create, Read, Update, Delete) completo para as entidades do estúdio, mas também garante a persistência de dados. O sistema implementa duas formas de "Front-End": uma Interface de Usuário de Texto (TUI) profissional, com cores e limpeza de tela, e um "Front-End de Consulta", que gera um relatório .html dos dados atuais. Um "Modo Demo" também foi implementado: se o programa iniciar sem arquivos .dat, ele cria dados fictícios para facilitar a demonstração.

O projeto utiliza CMake para um processo de compilação (build) moderno e multiplataforma, estando pronto para ser compilado e executado.

Applications (Como usar o programa)

Este projeto não é um site, mas sim um programa em C++ que roda no seu terminal. Para usá-lo, você precisa baixar o código-fonte e compilá-lo usando CMake (o comando g++ simples não funcionará devido à estrutura de pastas).

1. Clone o repositório:

git clone https://github.com/laissaraiva/StudioYoga-Manager-cpp.git

2. Entre na pasta e prepare a compilação (Build):

cd StudioYoga-Manager-cpp
mkdir build
cd build
cmake ..

3. Compile o projeto:

cmake --build .

4. Execute o programa:

./StudioYogaManager
